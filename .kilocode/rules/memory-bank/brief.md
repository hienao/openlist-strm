# OpenList to Stream 项目概述

## 项目基础

**项目名称**: OpenList to Stream (openlist-strm)

**项目类型**: 全栈Web应用程序

**主要功能**: 将OpenList文件列表转换为STRM流媒体文件的自动化工具

## 技术栈

### 前端技术栈
- **框架**: Nuxt.js 3.17.7 + Vue 3
- **样式**: Tailwind CSS
- **状态管理**: Pinia
- **构建工具**: Vite
- **运行时**: Node.js 20

### 后端技术栈
- **框架**: Spring Boot 3.3.9
- **数据访问**: MyBatis
- **任务调度**: Quartz Scheduler
- **数据库**: SQLite 3.47.1
- **数据库迁移**: Flyway
- **构建工具**: Gradle 8.14.3
- **运行时**: JDK 21

### 部署和运维
- **容器化**: Docker多阶段构建
- **Web服务器**: Nginx
- **认证**: JWT + Spring Security
- **API文档**: OpenAPI 3 (Swagger)

## 项目架构

### 整体架构
- **前后端分离架构**: 前端Nuxt.js + 后端Spring Boot
- **RESTful API**: 前后端通过JSON API通信
- **容器化部署**: Docker多阶段构建，单容器部署
- **数据持久化**: SQLite数据库存储配置信息
- **任务调度**: Quartz定时任务执行STRM文件生成

### 核心模块
1. **用户认证模块**: JWT令牌认证，支持注册、登录、密码修改
2. **OpenList配置管理**: 管理OpenList服务器连接配置
3. **任务配置管理**: 创建、编辑、删除STRM生成任务
4. **定时任务调度**: 基于Cron表达式的自动化任务执行
5. **媒体文件处理**: STRM文件生成和AI媒体信息刮削
6. **系统配置管理**: 日志级别、数据上报等系统设置

## 核心需求和目标

### 主要目标
构建一个将OpenList文件列表自动转换为STRM流媒体文件的全栈应用系统。该系统需要支持任务管理、定时执行、AI媒体刮削，并提供完整的Web管理界面。

### 核心功能需求
1. **STRM文件生成**: 自动将OpenList文件列表转换为STRM流媒体文件
2. **任务管理**: 支持创建、编辑、删除和手动执行转换任务
3. **定时调度**: 基于Cron表达式的定时任务自动执行
4. **增量更新**: 支持增量和全量两种更新模式，提高处理效率
5. **AI媒体刮削**: 集成AI技术，根据文件名自动识别和刮削媒体元数据
6. **用户认证**: 基于JWT的安全认证系统，保护管理界面
7. **容器化部署**: 完整的Docker支持，简化部署和运维

### 技术架构要求
- **前后端分离**: 现代化的前端框架 + 稳定的后端服务
- **RESTful API**: 标准化的API设计，支持第三方集成
- **数据库轻量化**: 使用SQLite，简化部署和维护
- **定时任务**: 可靠的任务调度机制，支持复杂的执行计划
- **扩展性**: 模块化设计，支持功能扩展和定制化
- **部署便捷**: 一键式容器部署，支持多种运行环境

### 用户体验要求
- **直观的Web界面**: 响应式设计，支持多设备访问
- **实时任务监控**: 任务执行状态实时显示和日志查看
- **配置管理**: 可视化的配置管理界面
- **错误处理**: 完善的错误提示和异常处理机制

## 项目特色

1. **自动化程度高**: 从OpenList获取文件列表到生成STRM文件全程自动化
2. **智能化处理**: 集成AI技术，智能识别媒体文件信息
3. **灵活的调度**: 支持复杂的定时任务配置，满足不同使用场景
4. **易于部署**: 完整的容器化解决方案，支持一键部署
5. **开源免费**: 基于GPL v3.0许可证，完全开源免费使用
